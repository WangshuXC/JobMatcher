## 重写腾讯招聘爬虫

### 关键发现
腾讯招聘有 JSON API，可直接获取结构化数据：
```
GET /tencentcareer/api/post/Query?categoryId=40001001,...&attrId=1&keyword=前端&pageIndex=1&pageSize=10&language=zh-cn&area=cn
```
返回：PostId、RecruitPostName、LocationName、BGName、ComName、Responsibility、SourceID 等。

### 列表页 URL
`https://careers.tencent.com/search.html?query=at_1,ot_40001001,ot_40001002,ot_40001003,ot_40001004,ot_40001005,ot_40001006&keyword=前端`

### 列表页 DOM 结构
- `.recruit-list` → 卡片容器
- `.job-recruit-title` → 标题
- `.job-recruit-location` → 地点
- `.recruit-tips` → 事业群 | 类别 | 经验 | 日期（span 列表，`.vertical-split-line` 分隔）
- `.recruit-text` → 描述摘要
- `.search-count` → "显示1-10,共32职位"
- 注意：卡片链接 href 为空，需通过 API 获取 PostId

### 详情页 DOM 结构（jobdesc.html?postId=xxx）
- `.job-recruit-title` → 标题
- `.job-recruit-location` → 地点
- `.recruit-tips` → 事业群/子公司信息
- `.duty.work-module .duty-text` → 岗位职责
- `.requirement.work-module .duty-text` → 岗位要求

### 实现策略
- **列表**: 通过 JSON API 获取，无需解析 HTML（效率更高、更稳定）
- **详情**: 对内部岗位（SourceID=1）访问详情页抓取"岗位要求"
- **外部链接**: SourceID=4 的岗位（如 PUBGM）指向 Workday，跳过详情抓取，直接使用 API 数据
- 前端分类 categoryId: `40001001,40001002,40001003,40001004,40001005,40001006`

## 探测字节跳动招聘 API

### 关键发现
字节跳动招聘也有 JSON API，且**不需要签名**，可以直接从服务端调用！

### 列表 API（POST）
```
POST https://jobs.bytedance.com/api/v1/search/job/posts
Content-Type: application/json

{
  "keyword": "前端",
  "limit": 10,       // 最大支持 50
  "offset": 0,       // 分页偏移
  "job_category_id_list": ["6704215862603155720"],  // 研发大类
  "tag_id_list": [],
  "location_code_list": [],
  "subject_id_list": [],
  "recruitment_id_list": [],
  "portal_type": 2,
  "job_function_id_list": [],
  "storefront_id_list": [],
  "portal_entrance": 1
}
```

返回字段：
- `data.count` - 总数（如 653）
- `data.job_post_list[].id` - 职位 ID
- `data.job_post_list[].title` - 标题
- `data.job_post_list[].description` - **完整职责描述**
- `data.job_post_list[].requirement` - **完整职位要求**
- `data.job_post_list[].city_info.name` - 城市
- `data.job_post_list[].city_list[]` - 多城市列表
- `data.job_post_list[].job_category.name` - 类别（如"前端"）
- `data.job_post_list[].job_category.parent.name` - 父类（如"研发"）
- `data.job_post_list[].recruit_type.name` - 类型（如"正式"）
- `data.job_post_list[].publish_time` - 发布时间戳
- `data.job_post_list[].code` - 职位编号

### 详情 API（GET）
```
GET https://jobs.bytedance.com/api/v1/job/posts/{postId}?portal_type=2&with_recommend=true
```
但因为列表 API 已包含完整 description 和 requirement，**实际不需要调详情 API**。

### 关键优势
- ✅ 不需要 `_signature`，直接调用
- ✅ 列表 API 返回完整数据（description + requirement），无需抓详情页
- ✅ `limit` 最大支持 50
- ✅ 比 Playwright 解析 DOM 快得多且更稳定
- category ID `6704215862603155720` = 研发大类

## 重写字节跳动爬虫（API 模式）

基于上述 API 探测结果，重写了 `src/lib/crawler/sources/bytedance.ts`：

### 实现策略
- **列表**: 通过 POST API 获取，API 已返回完整 description + requirement
- **详情**: `crawlJobDetail` 直接返回列表 API 的数据，不再访问详情页
- **分页**: 使用 offset/limit 模式，limit 设为 50（API 最大值）
- **session**: 仍需先访问首页建立 cookie，再在 page.evaluate 中调 fetch

### 对比旧版
| 项目 | 旧版（DOM 解析） | 新版（API） |
|------|-----------------|------------|
| 列表获取 | 等 SPA 渲染 → 解析 `a[href*="/position/"]` | POST API 直接获取 JSON |
| 详情获取 | 每个岗位都要打开详情页解析 DOM | 完全不需要，列表 API 已包含 |
| 抓取 10 个岗位耗时 | ~60s（10 次页面加载） | ~5s（1 次 API 调用） |
| 稳定性 | CSS class 可能变化 | 结构化 JSON，很稳定 |

### 验证结果
```

✅ crawlJobList(5): 返回 5 个职位，description 和 requirement 均完整
✅ 分页 offset=50: 正常返回不同的岗位
```

## 修复：抓取完职位后切到职位列表不实时更新

### 问题
爬虫抓取完职位后，用户切换到"职位列表" tab 时，列表不会刷新显示新数据。

### 根因
使用 `@base-ui/react/tabs`，其 `TabsPanel` 默认 `keepMounted=false`——非活跃 tab 的内容会被卸载。当抓取完成后 `refreshTrigger` 变化时，`JobList` 组件未挂载，无法响应。

### 修复
在 `src/app/page.tsx` 中将 `Tabs` 从非受控模式改为受控模式：
- 新增 `activeTab` state + `handleTabChange` 回调
- 使用 `value={activeTab}` + `onValueChange={handleTabChange}` 受控 Tabs
- 在 `handleTabChange` 中，当切到 `"jobs"` tab 时自动递增 `refreshTrigger`，确保 `JobList` 挂载后立即拉取最新数据

## 修复：职位列表切换公司筛选时闪动

### 问题
ScrollArea 不在顶部时，切换公司筛选按钮会导致列表闪动/跳跃。

### 根因
三个因素叠加：
1. `AnimatePresence mode="popLayout"` + `layout` 属性：旧卡片 exit 动画 + 新卡片 initial 动画 + layout 重排动画同时进行
2. exit 动画期间滚动容器内容高度突变，`layout` 对每个元素做位置插值
3. 每个卡片有 `delay: index * 0.03` 的渐进延迟，加剧闪动感

### 修复（`src/components/JobList.tsx`）
1. **移除 `AnimatePresence`** 和 `layout`/`exit` 动画——筛选切换不需要退场动画
2. **新增 `listKey` state**：筛选后递增 key，React 直接替换整个列表 DOM（不做 diff），新列表只有轻量的 fade-in 入场
3. **新增 `scrollRef` + `scrollToTop()`**：筛选后自动将 ScrollArea viewport 滚回顶部
4. 减小动画幅度：`y: 20 → 12`，`delay: 0.03 → 0.02`，`max delay: 0.5 → 0.3`
